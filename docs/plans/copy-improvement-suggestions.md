# Evame コピー改善提案

## 目的

Aboutページのコピーをマーケティング観点から分析し、コンバージョン向上のための改善案を提案する。

---

## 現状分析

### 強み

1. **明確なビジョン** - 「言葉の壁がないインターネット」というコンセプトが一貫している
2. **創業者ストーリー** - 個人的な動機（仏典を読みたかった）が具体的で共感を呼ぶ
3. **機能説明が簡潔** - 書く・届く・磨く・読むの4機能を短い言葉で伝えている
4. **多言語対応** - 5言語でローカライズされている

### 改善の余地

1. **ターゲットが不明確** - 誰向けのサービスかが伝わりにくい
2. **ベネフィットが弱い** - 機能説明はあるが「だから何が嬉しいのか」が薄い
3. **社会的証明がない** - ユーザー数、記事数、翻訳数などの実績がない
4. **CTAが弱い** - 「Start Writing」だけでは行動を促す力が弱い
5. **差別化ポイントが不明** - Google翻訳やDeepLとの違いが伝わらない

---

## 改善提案

### 1. ヒーローセクション

#### 現状
```
言葉の壁がないインターネット。
母国語で書く。世界が読む。
```

#### 提案A: ターゲット明示型
```
あなたの言葉が、世界に届く。
ブロガー、研究者、クリエイターのための多言語パブリッシングプラットフォーム。
```

#### 提案B: ベネフィット強調型
```
母国語で書くだけで、100言語の読者に届く。
翻訳の手間ゼロ。あなたは書くことに集中。
```

#### 提案C: 数字で訴求
```
言葉の壁がないインターネット。
すでに○○本の記事が、○○言語に翻訳されています。
```

**推奨:** 提案Bまたは提案C（実績が出てきたらCに移行）

---

### 2. 創業者ストーリー

#### 現状
```
なぜ作ったのか
読みたいものがあった。
ブッダの言葉。でも日本語の完訳は戦前のものしかない。言葉の壁で、読めなかった。
だから作った。言葉の壁がないインターネットを。
```

#### 評価
- **強み:** 具体的で共感を呼ぶストーリー
- **弱み:** ユーザーにとっての関連性が見えにくい

#### 提案: ユーザー視点を追加
```
なぜ作ったのか

読みたいものがあった。
ブッダの言葉。でも日本語の完訳は戦前のものしかない。
言葉の壁で、読めなかった。

だから作った。言葉の壁がないインターネットを。

あなたにも、届けたい言葉があるはずだ。
```

---

### 3. 機能説明（書く・届く・磨く・読む）

#### 現状の問題点
- 機能の説明になっていて、ユーザーのメリットが弱い
- 「磨く」の投票機能は差別化ポイントなのに説明が薄い

#### 提案

**書く（現状）**
```
母国語で書くだけ。翻訳の手間なし、書くことに集中。
```

**書く（改善案）**
```
書く

翻訳のことは考えなくていい。
いつもの言葉で、いつものように書くだけ。
```

---

**届く（現状）**
```
自動で翻訳。世界の読者に届く。
```

**届く（改善案）**
```
届く

公開した瞬間、○○言語に自動翻訳。
英語が書けなくても、世界中の読者があなたの記事を読める。
```

---

**磨く（現状）**
```
読者が投票で良い訳を選ぶ。時間が経つほど品質が上がる。
```

**磨く（改善案）** ← 最大の差別化ポイント
```
磨く

機械翻訳のままじゃない。
読者が「この訳がいい」と投票。人の目で、品質が磨かれていく。
Wikipediaのように、みんなで良くしていく翻訳。
```

---

**読む（現状）**
```
原文だけ、訳だけ、両方。切り替えは自在。語学学習にも、正確な理解にも。
```

**読む（改善案）**
```
読む

原文と翻訳を並べて読める。
「この訳、ちょっと違う」と思ったら、原文を確認。
語学学習にも、正確な理解にも。
```

---

### 4. 社会的証明の追加

#### 提案: 数字セクションを追加
```
○○本の記事
○○言語に翻訳済み
○○人のライター
```

または

```
「英語が書けないから諦めていた海外読者に、やっと届けられた」
— ○○さん（ブロガー）
```

---

### 5. CTAの強化

#### 現状
```
書く人がいて、読む人がいる。
言葉が違っても。

[Start Writing]
```

#### 提案A: 緊急性を追加
```
あなたの言葉を、世界に届けよう。

無料で始める →
```

#### 提案B: 具体的なアクション
```
最初の記事を、世界に届けよう。

今すぐ書き始める →
（登録不要・無料）
```

#### 提案C: 段階的CTA
```
まずは読んでみる →（ログイン不要）
書いてみる →（無料登録）
```

## ターゲット別メッセージング

### 想定ターゲット

1. **ブロガー・ライター** - 読者を増やしたい
2. **研究者・専門家** - 知見を広く共有したい
3. **語学学習者** - 原文と翻訳を比較して学びたい
4. **翻訳に関心がある人** - より良い翻訳を作りたい

### ターゲット別訴求

| ターゲット | 刺さるメッセージ |
|------------|------------------|
| ブロガー | 「読者が10倍に」「英語圏にリーチ」 |
| 研究者 | 「論文・知見を世界に」「引用されやすく」 |
| 語学学習者 | 「原文と翻訳を並べて読める」 |
| 翻訳好き | 「投票で良い訳を選ぶ」「翻訳を磨く」 |

---

## 競合との差別化

### 主な競合
- Google翻訳 / DeepL - 翻訳ツール
- Medium / note - パブリッシングプラットフォーム
- Wikipedia - 多言語コンテンツ

### Evameの独自価値
1. **パブリッシング + 翻訳の統合** - 書いたら自動で多言語展開
2. **コミュニティによる翻訳改善** - 機械翻訳を人が磨く
3. **原文と翻訳の並列表示** - 学習・検証に最適

### 差別化メッセージ案
```
Google翻訳で十分？

機械翻訳には限界がある。
Evameなら、読者が「この訳がいい」と投票。
時間が経つほど、翻訳の品質が上がっていく。
```

## 結論

Evameのコピーは「言葉の壁がないインターネット」という明確なビジョンを持っていますが、**ターゲットの明確化**と**ベネフィットの具体化**が改善ポイントです。

特に「磨く」機能（投票による翻訳改善）はユニークな差別化ポイントなので、もっと強く訴求すべきです。

また、サービスの成長に伴い、数字やユーザーの声を追加していくことで、信頼性と説得力が増していきます。

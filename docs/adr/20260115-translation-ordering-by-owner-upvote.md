# ADR: 翻訳の並び順でページオーナーの upvote を最優先する

- 日付: 2026-01-15
- ステータス: 採用
- 関連要件: `docs/requirements.md` の「翻訳の並び順」

## 背景
翻訳一覧は point の降順で表示していた。

## 問題
ページオーナーが良いと判断した翻訳が、必ずしも最上位に表示されない。

## 判断
ページオーナーが upvote した翻訳を最上位に固定し、それ以外は point 降順、同点は createdAt 降順とする。

## 代替案
- point のみで並べる（現状維持）
- 専用の「ピン留め」機能を追加する

## 影響
- 翻訳一覧 API の並び順が変更される
- UI 側は API の順序を信頼できるようになる

## 実装メモ（任意）
- ページオーナーの upvote 判定は translation_votes を参照する
